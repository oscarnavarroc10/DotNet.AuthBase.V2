# Vision

## Project Name

DotNet.AuthBase.V2

## Purpose

DotNet.AuthBase.V2 is a reusable, production-ready authentication foundation for ASP.NET Core applications. It provides a secure, well-tested, and extensible base that teams can integrate or fork to accelerate the delivery of authenticated APIs and web applications.

## Goals

- Provide a reference implementation of authentication best practices for ASP.NET Core.
- Support JWT-based authentication with refresh token rotation.
- Ensure all features are thoroughly specified and tested before delivery.
- Enable AI-assisted development through structured, Spec-Driven workflows.
- Remain modular so individual components can be adopted independently.

## Non-Goals

- This project does not aim to replace identity providers such as Auth0, Okta, or Azure AD.
- It does not provide a UI or front-end application.
- It does not manage authorization policies beyond basic role-based access control.

## Success Criteria

- All authentication flows are backed by approved specifications.
- 80%+ test coverage across all production code.
- Security review completed for every authentication-related component.
- Documentation is sufficient for a new engineer to understand and contribute within one hour.
